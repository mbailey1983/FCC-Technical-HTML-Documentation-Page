Introduction to HTML

"At its heart, HTML is a fairly simple language made up of elements, which can be applied to pieces of text to give them different meaning in a document (Is it a paragraph? Is it a bulleted list? Is it part of a table?), structure a document into logical sections (Does it have a header? Three columns of content? A navigation menu?), and embed content such as images and videos into a page. As HTML has evolved, it also has become more accessible to people with a variety of disabilities." 1

HTML is a fairly easy programming language to learn. It doesn't require complex varibles and is actually very intuitive. This documentation will give you the basics to begin coding webpages in HTML. This will not be comprehensive but will get you started.

  • HTML uses elements to mark sections of website content.
  • HTML elements are represented with tags
  • "HTML tags label pieces of content such as "heading", "paragraph", "table", and so on"
  • "Browsers do not display the HTML tags, but use them to render the content of the page"

Setting Up A Webpage

A webpage has to have a framework before you can begin adding content. The framework allows you to divide the webpage up into major sections, such as the page title, header, body, and footer. To see how this framework is set up on every webpage, take a look at the code below.

<!DOCTYPE html> <html> <head> <title>Page Title</title> </head> <body> <!-- Put your website content here --> </body> </html>

Let's break this down. First, we have the ❮!DOCTYPE html❯ tag. This let's the browser know that the webpage is written in html. With all the new coding languages available for websites, such as javascipt, ruby, etc, it is vital to let the browser know what to expect. The main tag that defines a webpage is the ❮html❯ tag. This both opens and closes the page. Next, we have the ❮head❯ tag. This part of the website is different from the header. The ❮head❯ section contains important information about your website, such as the site title that will appear in the browser tab, any stylesheets or scripts that need to run on the website, and Facebook meta data. Visitors won't actually see this content, but it directly affects how your website looks and runs.

After that, we have the ❮body❯ tag, which opens and closes the main website content. This is where your menu, copy, images, and other content will be added. Everything in this section of the website will be seen by your visitors. This will even include the footer section of your website. These tags are crucial to a functioning webpage. In fact, it is best to save this barebones template as a starting point for any coded webpage you build. Please note that every tag in HTML, apart from a line break, must have a closing tag to be complete. You can see this in the above code where each tag is closed in this format: ❮/body❯.

Fonts

A website is pretty bare without some text. In this section, you'll see the various tags used for adding text and images to your webpage.

Text is added to webpages by using heading and paragraph tags. It can also take different tags to change the style of text, such as italics or bold. The list of options includes:

  • Heading tags: sizes 1-6 with 1 being the largest
  • Paragraph tag: starts a new paragraph in baseline font
  • Decoration tags: allows for formatting text with italics, bold, underline, superscript, subscript, etc

Let's see how this works in our code example from above.

<!DOCTYPE html> <html> <head> <title>Page Title</title> </head> <body> <h1>My Awesome Website</h1> <p>Please read my website. I have such <em>valuable</em> information. This includes <b>ideas</b> and <u>tips</u>.</p> <h3>What do you think?</h3> </body> </html>

The above code renders like this on a live website:

My Awesome Website

Please read my website. I have such valuable information. This includes ideas and tips.

What do you think?

Feel free to play around in a code playground to see how different font tags affect your text!

Images

Images are equally important when creating a webpage. Images convey meaning, help develop your topic, and provide a shareable item in today's visual world. But, unlike text, which you can add directly into a webpage, images must be stored somewhere and pulled into your webpage using it's URL. Most webpages that are hosted will have storage for files, including images. Otherwise, for testing and learning, you can store any images in Dropbox or similar cloud service. The only requirement is to make sure you can get the direct image link that ends with the file type, like .jpg or .png, by right clicking the image and then clicking Copy Image Address

We are going to add an image to our existing webpage. This will use the tag ❮img❯. However, it isn't enough to simply put that tag in. We also have to declare the source, abbreviated src, of the image. It is also highly recommended that you give the image an alternative name. The alt attribute gives both search engines and screen readers a text description of the image. Let's see an example of our webpage with the added image below.

<!DOCTYPE html> <html> <head> <title>Page Title</title> </head> <body> <h1>My Awesome Website</h1> <img alt="screaming llama" src="https://www.dropbox.com/s/nk7prog3oxumx3z/5688248840_8f31d0acdf_b.jpg?dl=0"> <p>Please read my website. I have such <em>valuable</em> information. This includes <b>ideas</b> and <u>tips</u>.</p> <h3>What do you think?</h3> </body> </html>

The above code renders like this on a live website:


My Awesome Website

Please read my website. I have such valuable information. This includes ideas and tips.

What do you think?

With the ❮img❯, you can add images to your webpage.

Conclusion and References

This documentation is very simple. There are many more tags, attributes, and more that can be used on a webpage. Also, this does not include styling that can be provided using CSS, or cascading style sheets. The majority of the colors, alignment, layout, and interactive elements in a webpage today are created using CSS. In fact, you can view HTML as the frame on the house and CSS as the way to paint the walls and add lights and carpet to the house.

With sufficient practice of the above basic skills, you will be prepared to tackle more difficult HTML coding. Don't give up!
